# Graph & Dynamic Programming Problem Set

## Setup

You have three problems of increasing difficulty in `problems.rb`. Each problem has a method stub with a docstring describing the expected behavior, inputs, and outputs.

## Your task (60 minutes total)

Work through all three problems in order. For each:

1. **Analyze** the problem (2-3 minutes): identify the pattern, think about edge cases
2. **Implement** a working solution
3. **Test** against the provided examples and your own edge cases
4. **Optimize** if your first solution is not optimal. State the time and space complexity.

### Time allocation

| Problem | Difficulty | Target Time | Pattern |
|---|---|---|---|
| 1. Course Schedule | Medium | 15 minutes | Graph (topological sort) |
| 2. Word Break | Medium | 15 minutes | Dynamic programming |
| 3. Cheapest Flights | Hard | 25 minutes | Graph + DP (Bellman-Ford variant) |

Reserve 5 minutes at the end to review and discuss trade-offs.

## Evaluation Criteria (Staff Level)

- **Correctness**: Does it handle all edge cases (empty input, cycles, unreachable states)?
- **Optimal complexity**: Is the time/space complexity optimal or near-optimal?
- **Code quality**: Clean variable names, no unnecessary complexity, idiomatic Ruby
- **Trade-off articulation**: Can you explain why you chose this approach over alternatives?
- **Testing instinct**: Do you proactively check edge cases without being prompted?

## Scoring

| Score | Description |
|---|---|
| Strong hire | All 3 solved optimally, clean code, articulated trade-offs |
| Hire | 2 solved optimally + reasonable approach on 3rd |
| Lean hire | 2 solved with minor bugs, discussed approach for 3rd |
| No hire | Fewer than 2 solved, or solutions have fundamental issues |
